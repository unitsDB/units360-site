Platform Overview
🚀 What is Units360?

Units360.ai is a low-code usage metering platform — the operating system for any business that sells or rewards by the unit. Instead of building custom ledger infrastructure, you define your unit model once, and Units360 auto-provisions everything else.

In this demo, the unit is reward_points. But it works equally for km, kWh, tokens, API calls, consultation hours — any measurable quantity.

⭐ What is a Unit?

A Unit is the fundamental thing being measured — in this case, reward_points. It defines what you're counting, which direction it flows, and when it resets.

unit_idreward_points
display_nameApex Reward Points
directionBIDIRECTIONAL — can earn (positive) and redeem (negative)
reset_periodNEVER — points accumulate until redeemed or expired
precisionInteger (whole points per transaction)

Core Concept · 2 of 3
💳 What is an Account?

An Account is the entity whose usage is tracked — here, a specific cardholder. Each account maintains a live points balance and its own rate plan.

account_idAPEX-4291 — cardholder identifier
card_typeApex Rewards Gold · premium tier
rate_planApex-Gold-1x — tiered by merchant category
current_balanceLive points total — updated per transaction
tier_statusGOLD → GOLD ELITE at 10,000 pts

Multiple cardholders can use the same reward_points unit but have different multipliers based on card tier, spending category, or promotional campaigns.

Core Concept · 3 of 3
📋 What is a Transaction?

A Transaction is a single card purchase or redemption event — it carries the spend amount, merchant category, and multiplier. Points are calculated and posted instantly.

quantity+350 pts — from a $35 dining purchase at 3×
categoryDINING — triggers 3× bonus multiplier
multiplier3× applied automatically from FMT rate rules
balance_afterCumulative points total updated after each txn
directionEARN (positive) or REDEEM (negative)
At redemption, a negative-quantity transaction debits points from the ledger. Units360 enforces balance guards — a redemption that would take balance below zero is rejected automatically.
Configuration Layer
📐 FMT — Fintech Metadata Template

The FMT is Units360's low-code config for loyalty programs. Define the entire points model once — earn rates, multipliers, tier thresholds, redemption rules. No code, no schemas.

Unit tabDefine reward_points, direction (bidirectional), precision, expiry
Account tabCard tier, base rate plan, tier thresholds, redemption floor
Transaction tabMCC-based multipliers, promo rules, balance guards
FMT outputs: Once saved, Units360 auto-generates a REST endpoint for transaction ingestion, a live points dashboard, tier-progress API, and a statement template. The card network sends authorization events — Units360 handles everything else.
Runtime Engine
⚙️ RTE — Run Time Engine

The RTE is the live processing layer. It ingests transaction events, resolves multipliers, updates the points ledger atomically, and triggers tier-change notifications in real time.

Txn ingestionAccepts CloudEvents from card network via REST POST
Multiplier resolverReads MCC code → applies matching rate rule from FMT
Points ledgerAtomic balance update per transaction — millisecond-accurate
Tier monitorWatches balance thresholds, fires tier-upgrade webhooks
Redeem guardRejects over-redemption; enforces minimum balance rules

Event Protocol
📡 CloudEvents — The Transaction Payload

Units360 uses the CloudEvents 1.0 spec. Every card transaction is formatted as a CloudEvent JSON payload and POSTed to the Units360 RTE endpoint.

// Card Network → Units360 RTE (one transaction) POST /api/v1/events/APEX-4291 Content-Type: application/cloudevents+json { "specversion": "1.0", "type": "com.units360.loyalty.earn", "source": "card/APEX-4291", "time": "2024-06-21T10:42:18Z", "data": { "unit": "reward_points", "qty": 350, "direction": "EARN", "amount_usd": 35.00, "mcc": 5812, "category": "DINING", "multiplier": 3 } }
data.unitMust match FMT unit_id — "reward_points" here
data.qtyPoints earned or redeemed this transaction
data.directionEARN or REDEEM — RTE applies balance guard on REDEEM
data.multiplierApplied by RTE from FMT rules — 1×, 2×, or 3× here
