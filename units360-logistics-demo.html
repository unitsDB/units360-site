Platform Overview
🚀 What is Units360?

Units360.ai is a low-code usage metering platform — the operating system for any business that sells by the unit. Instead of building custom billing infrastructure, you define your unit model once, and Units360 auto-provisions everything else.

In this demo, the unit is km (kilometer). But it works equally for kWh, tokens, API calls, consultation hours — any measurable service.

Core Concept · 1 of 3
📍 What is a Unit?

A Unit is the fundamental thing being measured and billed — in this case, kilometers driven. It defines what you're counting, how it accumulates, and when it resets.

unit_idkm — the unique identifier
display_nameKilometer
directionCUMULATIVE — always increases, never negative
reset_periodPer-Route — balance resets at route completion
precision1 decimal place (e.g., 0.4 km per GPS ping)
km · miles · hours API calls · tokens kWh · gallons · credits
Core Concept · 2 of 3
🚛 What is an Account?

An Account is the entity whose usage is tracked — in this case, a specific vehicle on a specific route. Each account gets its own running distance balance governed by its rate plan.

account_idTRK-CA-10023 — vehicle identifier
typeBox Truck · 20-tonne capacity
rate_planStandard-Fleet-KM — tiered by segment type
cargo_classStandard Parcel (≤30 kg per unit)
current_balanceLive km total — updated per GPS ping

Multiple trucks can share the same km unit definition but have different rate plans based on vehicle class, cargo type, or shipper agreements.

Core Concept · 3 of 3
📋 What is a Transaction?

A Transaction is a single GPS ping — one reading that says "this vehicle traveled X km since the last ping." Transactions build the running distance total and trigger pricing.

quantity0.47 km — distance since last ping (30s interval)
segmentHIGHWAY — determines which rate tier applies
price_applied$0.42 base + $0.08 fuel surcharge = $0.50/km
balance_afterCumulative km total updated after each ping
timestampISO 8601 UTC — every ping is precisely timestamped
At route completion, all transactions are summed by segment type. Highway km get the base rate, urban km get the premium surcharge. The invoice is generated automatically — no manual calculation.
Configuration Layer
📐 FMT — Fleet Metadata Template

The FMT (Fleet Metadata Template) is Units360's low-code config builder for fleet operators. Define the metering model once — no code, no schemas, no infrastructure.

Unit tabDefine km, direction (cumulative), precision, reset period
Account tabVehicle type, rate plan, cargo class, GPS ping interval
Transaction tabBase rate per km, fuel surcharge, urban premium, segment rules
FMT outputs: Once saved, Units360 auto-generates a REST endpoint for GPS ingestion, a real-time route dashboard, and an invoice template. The carrier's dispatch system sends GPS pings — Units360 handles everything else.
Runtime Engine
⚙️ RTE — Run Time Engine

The RTE is the live processing layer. It ingests GPS events, classifies route segments, applies pricing rules, and maintains the distance ledger — all in real time.

GPS ingestionAccepts CloudEvents from vehicle GPS units via REST POST
Segment classifierDetects highway vs urban based on speed + geofence
Distance ledgerAtomic cumulative km update per ping — millisecond-accurate
Rate applicationReads FMT rules; applies base, surcharge, or urban premium per segment
Invoice triggerOn delivery scan event → auto-generates route invoice JSON

Event Protocol
📡 CloudEvents — The GPS Payload

Units360 uses the CloudEvents 1.0 spec. Every GPS ping the vehicle sends is formatted as a CloudEvent JSON payload and POSTed to the Units360 RTE endpoint.

// Vehicle GPS → Units360 RTE (one ping) POST /api/v1/events/TRK-CA-10023 Content-Type: application/cloudevents+json { "specversion": "1.0", "type": "com.units360.fleet.gps", "source": "gps/TRK-CA-10023", "time": "2024-06-21T09:42:18Z", "data": { "unit": "km", "qty": 0.47, "speed_kmh": 94, "heading": "SE", "odometer": 287.4, "segment": "HIGHWAY" } }
data.unitMust match FMT unit_id — "km" here
data.qtykm driven since last ping — added to running total
data.speed_kmhCurrent speed — used for segment classification
data.segmentHIGHWAY or URBAN — determines pricing tier
